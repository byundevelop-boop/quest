# AI Assist 프롬프트 기록 (제출용)

## 1) 진행 방식 요약
`상세 리서치 -> 플랜 수립 -> 요구사항 세분화/구현 -> 리뷰 반복 -> 테스트 검증`

## 2) 단계별 프롬프트 로그

### Step 1. 상세 리서치 (Deep Research)
목표: 구현 전에 요구사항을 깊게 파악하고 누락 리스크를 제거.

```text
컬리 과제 요구사항을 디테일하게 전부 분석해줘.
UI 규칙(타입별 줄수/가격표시), API 스펙, 페이징, 찜하기, 새로고침,
예외 상황까지 빠짐없이 정리하고 모호한 지점은 리스크로 표시해.
```

```text
요구사항을 완료 체크 기준으로 바꿔줘.
각 항목마다 검증 방법(어떤 테스트/확인으로 통과인지)까지 붙여줘.
```

산출물:
- `exam_materials/컬리_과제_상세분석.md`

### Step 2. 플랜 수립 (Architecture Plan)
목표: 구현 제약을 먼저 잠그고 설계부터 고정.

```text
클린아키텍처 + 멀티모듈로 설계해줘.
단, mockserver 모듈은 절대 수정하지 말고 연동만 하도록 제약을 걸어.
domain/data/presentation 책임과 의존성 방향을 명확히 해줘.
```

```text
domain 계약(Repository/UseCase/Policy)을 먼저 확정하고,
data/presentation은 그 계약을 따르도록 구현 순서를 제안해줘.
```

핵심 제약:
- `mockserver`는 수정 금지
- 멀티 모듈 유지
- 의존성 방향 고정 (`presentation -> domain -> data`)

### Step 3. 요구사항 잘게 쪼개기 + 구현
목표: 큰 작업을 작은 단위로 쪼개서 실수 없이 진행.

```text
남은 작업을 TODO 체크리스트로 아주 잘게 쪼개줘.
한 번에 1개만 진행하고, 완료 즉시 체크하도록 운영 규칙도 같이 써줘.
```

```text
TODO에서 다음 1개만 구현해.
끝나면 체크박스를 [x]로 바꾸고 변경 파일/핵심 동작만 요약해.
```

운영 파일:
- `docs/TODOLIST.md`

### Step 4. 리뷰 반복 (Review, Review, Review)
목표: 요구사항 누락, 회귀, 테스트 공백을 반복 점검.

```text
코드리뷰 에이전트 모드로 점검해줘.
요구사항 누락 -> 회귀 위험 -> 테스트 공백 순으로 우선순위를 두고 리뷰해.
수정 후 다시 같은 기준으로 재검토해.
```

반복 규칙:
- 누락 발견 시 즉시 수정
- 수정 후 동일 항목 재리뷰
- TODO 상태 업데이트

### Step 5. 테스트/검증
목표: 최종 제출 전 실수 여부를 명령어 기반으로 검증.

```text
최종 검증 커맨드를 실행해.
테스트 성공/실패, 빌드 성공/실패, 산출물 경로까지 보고해.
```

## 3) 컨텍스트 부족 시 운영 프롬프트
```text
컨텍스트가 부족해지면 docs/TODOLIST.md를 먼저 갱신해.
완료 항목은 체크 유지, 미완료 항목 중 다음 1개만 지정해.
다음 컨텍스트에서 그 1개부터 재개하도록 handoff 문구를 남겨줘.
```

## 4) 제출용 완료 보고 문구
```text
처음에 과제 요구사항을 상세하고 깊게 리서치한 뒤,
클린아키텍처 + 멀티모듈 플랜을 수립해 구현했습니다.
요구사항은 TODO 단위로 잘게 쪼개서 하나씩 완료 체크하며 진행했고,
코드리뷰를 반복 수행해 누락/회귀/테스트 공백을 지속 점검했습니다.
마지막으로 테스트와 빌드를 통해 실수 여부를 검증했고 결과물까지 확인했습니다.
```
