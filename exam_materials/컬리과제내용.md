## 요구사항 1

### 1) UI

- 첨부자료 참고(`과제_UI.png`)
- 마진/패딩 등 미적인 관점 요소는 고려하지 않음

### 2) 메인 기본

- 메인은 여러 개 섹션으로 구성

### 3) 섹션

- `sections` API에서 페이지 정보 + 섹션 정보 리스트 제공
- 섹션 타입 3개:
- `vertical`: 세로 스크롤 가능한 섹션
- `horizontal`: 가로 스크롤 가능한 섹션
- `grid`: 3(col) x 2(row), 총 6개 상품 노출 섹션
- 각 섹션마다 `title` 별도 노출 필요
- 한 섹션마다 여러 상품 포함

## 요구사항 2

### 4) 상품

- `section/products` API에서 해당 섹션 상품 목록 제공
- 상품 뷰 모양:
- `vertical`, `grid/horizontal` 형태가 다름
- 제목 규칙:
- `horizontal`, `grid`: 2줄 + 말줄임
- `vertical`: 1줄 + 말줄임
- 가격 규칙:
- 할인 있을 경우: 할인율(`#fa622f`), 할인가격(`bold`), 원래가격(`취소선`)
- 할인 없을 경우: 원래가격만 노출
- `horizontal`, `grid`: 첫 줄(할인율 + 할인가), 둘째 줄(원래가격)
- `vertical`: 한 줄에 모든 가격정보 노출
- 자세한 UI는 `과제_UI.png` 참고
- 모든 상품은 찜하기 가능

### 5) 찜하기

- 서버 없이 구현
- 찜하기 여부에 따라 아이콘 다르게 노출

### 6) 새로고침

- 메인 화면 pull-to-refresh 제공

### 7) 서버 제약조건

- 별도 모듈에서 제공하는 `MockInterceptor` 사용
- 프로젝트는 multi-module 구성 필요

## 참조 1. sections API

### 1) 섹션 목록 불러오기

- Endpoint: `sections`
- Query: `page: Int`
- `type`에 따라 뷰타입 분기, `id`에 따라 각 섹션 상품목록 노출
- 첨부자료 참고: `sections_1.json`

## 참조 2. section products API

### 1) 섹션 별 상품 목록

- Endpoint: `section/products`
- Query: `sectionId: Int`
- 첨부자료 참고: `section_products_7.json`

## 참조 3. 찜하기 이미지 파일 / Mock Server API 사용법

### 1) 찜하기 이미지 파일

- 첨부자료 참고:
- `ic_btn_heart_on.xml`
- `ic_btn_heart_off.xml`

## FAQ

### Q. 기능을 임의로 추가, 구현하면 가산점이 있나요?

A. 작업 방식, 언어 숙련도, 코딩 스타일, AI 활용 등 기본 항목 중심으로 검토 예정입니다.

### Q. 사전과제 제출 방법이 궁금합니다.

A. GitHub 저장소를 Public으로 전환한 뒤 링크 전달하면 됩니다.  
Private 저장소로 진행 희망 시 채용담당자에게 별도 문의가 필요합니다.

### Q. GitHub 대신 Bitbucket, GitLab 등을 이용해도 되나요?

A. 네, 확인 가능하면 됩니다. Issue 관리도 다른 툴 사용 가능합니다.

### Q. 사전과제 기한은 어떻게 되나요?

A. 기본 안내는 약 5일입니다. 일정 조율이 필요하면 채용 담당자와 시작/종료 일자 조율 가능합니다.
